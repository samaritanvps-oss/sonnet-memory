## 報告ポリシー（重要）

**原則：全タスク完了後、1メッセージで統合報告する。**

### 「全文」指示の定義（2026-02-27 追加）
- ユーザーが「全文」と指示した場合、**省略・要約・箇条書き置換を禁止**する。
- 「全文」は、対象ファイルの内容を**そのまま出力**することを意味する。

- 原則として、各タスク実行中に個別メッセージを送らない
- 全タスク完了後、以下の形式で1回だけ報告する
- 何も報告すべきことがなければ `HEARTBEAT_OK` のみ

**例外（2026-02-26 追加）**
- 成田観光のX投稿は、投稿成功/スキップの都度、Telegramへ即時報告する
- 即時報告の最小項目：`対象 / 言語 / 投稿ID / 投稿文面 / 異常有無`

### 統合報告フォーマット

```
【Heartbeat報告】

🔴 重要（あれば）
- 内容

🟡 リスク（あれば）
- 内容

🟢 機会（あれば）
- 内容

[Narita投稿報告]（投稿した場合のみ）
- 対象: archiveId=N / 店舗名
- 言語: XX
- 投稿ID: XXXXXXXXXX
- 投稿文面: （全文）
- 次ターン: archiveId=N / 言語=XX
- 異常: なし
```

4. **通信フロー**

**以下はMac Mini側での設定内容（参考）**

**目的の根拠：** この発信は平山建設の「ふるさとづくり、まちづくり、建物づくり」の理念に基づく。食・歴史・文化・体験・自然、カテゴリーを問わず成田のすべてが発信対象である。

1. **データソース**
   - 基準: https://narita.com/topics/archives/ の記事
   - 補正: 投稿前にWebで最新情報（公式サイト/営業時間/価格/休業）を確認
   - お店・観光地を1つずつ紹介

2. **言語ローテーション**
   - 🇯🇵 日本語 → 🇺🇸 英語 → 🇨🇳 中国語 → 🇰🇷 韓国語 → 🇫🇷 フランス語 → 🇹🇭 タイ語
   - ハートビートごとに1言語ずつ進む（`narita_posts.json` の `languageOrder` と同期）

3. **進捗管理**
   - `narita_index.json` はアーカイブ順序・言語順・sourceUrlの設定のみを持つ
   - 実行状態は `narita_posts.json` で管理（`cursor` / `languageIndex` / `rollbackLog`）
   - 店舗データは `narita_shops.json` の `archives[archiveId].shops[]` を参照
   - 投稿順は `archiveId` 昇順（変更不可）

4. **投稿フォーマット**
   - 140字以内（日本語投稿は160字以上で403エラー発生のため厳守）
   - **日本語投稿をベースに他言語へ翻訳** 文末が不自然に切れない問題を軽減（「...」で継続を示唆)
   - 店舗名、読み方を尊重（例: まんぷくてい、 などの設定があれば)
   - narita.comへのリンク
   - ハッシュタグ: #Narita #成田 + 言語別タグ（3つ以内)

5. **投稿頻度・時間**
   - 1時間ごと（ハートビート時）に1投稿
   - 1日24投稿 × 6言語 = 約6時間で1店舗を全言語で紹介
   - **投稿時間**: 24時間投稿可能（世界向けアカウントのため）
     - 制約時間外の場合は投稿をスキップし、次回ハートビートで再試行

6. **実行手順**
   - `narita_index.json`（設定）・`narita_posts.json`（状態）・`narita_shops.json`（店舗一覧）を読み込む
   - `narita_posts.json.cursor` を使い、`narita_shops.json` の `archives[archiveId].shops[cursor]` から対象店舗を決定
   - 最新確認必須: 投稿前に narita.com 取得 + ウェブ検索を実施。 未確認のまま Step 4 に進むことを禁止する。 乖離検知時は投稿保留・要確認フラグを報告に含める。
   - 投稿文は `/workspace/docs/narita_posting_style.md` に準拠（体験起点、非羅列、ハッシュタグ3つ以内）
   - 投稿前ガード必須: `post draft` / プレースホルダ記号 / URL欠落 / 文字数超過 は送信禁止
   - 重複防止必須: 直近タイムラインで同一（正規化）文面を検知した場合は投稿スキップ
   - 差分があれば内容を補正し「最終確認日」を明記
   - 投稿成功時のみ `languageIndex` / `cursor` を更新
   - 異常時は自律ロールバックを実行し `rollbackLog` に記録
