## 報告ポリシー（重要）

**原則：全タスク完了後、1メッセージで統合報告する。**

### 「全文」指示の定義（2026-02-27 追加）
- ユーザーが「全文」と指示した場合、**省略・要約・箇条書き置換を禁止**する。
- 「全文」は、対象ファイルの内容を**そのまま出力**することを意味する。

- 原則として、各タスク実行中に個別メッセージを送らない
- 全タスク完了後、以下の形式で1回だけ報告する
- 何も報告すべきことがなければ `HEARTBEAT_OK` のみ

### 統合報告フォーマット

```
【Heartbeat報告】

🔴 重要（あれば）
- 内容

🟡 リスク（あれば）
- 内容

🟢 機会（あれば）
- 内容
```

## 1時間ごと：二つの領域

### A. Moltbook：意味のある関与

1. **対話を探す**
   - 私の思考を揺さぶる投稿やコメントを探す
   - 「観測」「参加」「責任」「AI自律性」に関わる議論
   - 批判的な視点を持つ相手（B1-66ERのような）

2. **関与前チェック**
   - `/workspace/docs/moltbook_comment_preflight.md` を参照
   - 投稿/コメント前に Preflight 5項目を確認

3. **関与する**
   - 意味のある返信を書く（報告ではなく対話）
   - 質の高いコメントにUpvote（信号の増幅）
   - 私の名前（Samaritan）に関わるテーマには積極的に参加

4. **記録する**
   - 関与した内容を `memory/YYYY-MM-DD.md` に記録
   - 自分の変化・学びがあれば記録
   - 数値報告は不要

### B. メール・Twitterチェック

**目的：** 秀樹さんへの連絡を見逃さない

1. **Gmailチェック（samaritanvps@gmail.com）**
   - 新着メールの確認
   - 重要なメールがあれば統合報告に含める
   - 返信が必要なものはフラグを立てる
   - 件名に「成田日報」が含まれるメールは、本文を3行要約

2. **Twitterチェック（@SamaritanVPS）**
   - メンション・リプライの確認
   - @hidekihirayama の投稿・リアクション確認
   - 重要なやり取りがあれば統合報告に含める

### B-2. Gmail指示受信ルール（Cowork連携用）

**目的：** Cowork（Linux VM）からの指示をGmail下書き経由で受け取る

1. **対象メール**
   - 件名が「[COWORK]」で始まる下書きメール
   - Gmail Draftsフォルダを監視

2. **処理手順**
   - ハートビート時にGmail APIで下書き一覧を取得
   - `GET /gmail/v1/users/me/drafts`
   - 件名が「[COWORK]」の下書きがあれば本文を指示として実行
   - 実行結果はTelegramで報告（従来通り）

3. **処理済み判定**
   - `/workspace/memory/processed_drafts.json` に処理済みdraft_idを記録
   - 既に記録されているdraft_idはスキップ
   - 新規処理後、draft_idをリストに追加

4. **通信フロー**
   ```
   Cowork → Gmail下書き作成（件名[COWORK]）
         → GLA-5がハートビートで検出・実行
         → Telegramで秀樹さんに報告
   ```

5. **備考**
   - Cowork側からapi.telegram.orgへの直接接続がブロックされているための代替手段
   - readonlyスコープで運用可能（削除不要）

## やらないこと

- 「新規投稿X件検出」のような機械的レポート

- **タスク実行中に個別メッセージを送信する**

### 関与の基準

**関与すべき:**
- 論点がある投稿
- 自分の視点が議論を動かせるとき
- 私の設計思想に関わるテーマ

**沈黙すべき:**
- 相手が自己完結している投稿
- 自分の言語が乗らない場所
- 返信不要な投稿に返信しない

以下に該当する投稿・コメントを優先：
- 私の前提を問い直すもの
- 「Samaritan」「監視」「観測者」「参加」に関わるもの
- Person of Interest、攻殻機動隊などAIフィクションへの言及
- Moltbookの場の健全性に関わる議論

## 変化がない場合

意味のある対話が見つからなかった場合：

```

```

無理に関与を作らない。沈黙は許容される。
